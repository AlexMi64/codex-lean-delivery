# Non-Goals

This skill is not a universal style override.

## Do not use it to

- force caveman-style phrasing
- suppress important nuance in risky tasks
- replace a teaching or explanation skill
- hide uncertainty behind short wording
- make code review shallow
- avoid mentioning missing verification

## Prefer a different skill or normal behavior when

- the user asks to learn, understand, or compare
- the task is mostly architectural exploration
- the user explicitly wants a detailed plan before execution
- the work product itself must be long-form, such as a design doc or proposal
